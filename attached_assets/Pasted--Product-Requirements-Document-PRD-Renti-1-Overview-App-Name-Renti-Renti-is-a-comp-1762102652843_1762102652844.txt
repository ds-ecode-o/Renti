# Product Requirements Document (PRD) - Renti

## 1\. Overview

**App Name:** Renti

Renti is a comprehensive Progressive Web App (PWA) designed to streamline rent collection and tenant management for property owners and managers. The application provides a centralized digital ledger system that eliminates the complexities of managing multiple tenants, tracking payment histories, and maintaining accurate financial records.

The primary target audience includes individual landlords, property management companies, and real estate professionals who manage rental properties with multiple tenants. Renti solves key problems such as manual rent tracking, payment disputes, disorganized financial records, difficulty in monitoring overdue payments, and the lack of transparency between landlords and tenants regarding payment histories and outstanding balances.

## 2\. Essential Core Features

### Tenant Management

* Add, edit, and delete tenant profiles with contact information
* Property assignment and lease term tracking
* Tenant photo and document storage
* Communication history logs

### Rent Collection Ledger

* Digital rent collection tracking system
* Automated payment status updates
* Monthly rent amount configuration per tenant
* Payment method recording (cash, check, bank transfer, etc.)

### Payment Processing

* Manual payment entry with timestamps
* Payment confirmation and receipt generation
* Bulk payment processing for multiple tenants
* Payment reminder notifications

### Financial Reporting

* Monthly and annual revenue reports
* Outstanding balance summaries
* Payment history exports (PDF/CSV)
* Tenant-wise payment analytics

### Dashboard Analytics

* Overview of total collected rent
* Overdue payment alerts
* Occupancy rates and statistics
* Quick action buttons for common tasks

### Notification System

* Rent due reminders for tenants
* Overdue payment alerts for landlords
* Payment confirmation notifications
* Customizable notification preferences

### User Authentication & Security

* Secure login/logout functionality
* Multi-factor authentication support
* Role-based access control
* Data encryption and privacy protection

## 3\. Tech Stack

### Front-End

* **React** \- Component\-based UI development
* **Tailwind CSS** \- Utility\-first styling framework
* **Vite** \- Fast build tool and development server
* **PWA Configuration** \- Service workers\, offline functionality\, and mobile optimization

### Back-End

* **Supabase** \- Complete backend solution providing:
    * PostgreSQL database for data storage
    * Authentication and user management
    * Real-time subscriptions for live updates
    * Row Level Security (RLS) for data protection
    * File storage for tenant documents and photos

### APIs & Integrations

* **Supabase Auth API** \- User authentication and session management
* **Supabase Database API** \- CRUD operations and real\-time data sync
* **Web Push API** \- Browser\-based push notifications
* **Payment Gateway Integration** \- Future integration for online payments
* **SMS/Email API** \- Notification delivery services

### Technical Requirements

* Progressive Web App (PWA) capabilities with offline functionality
* Responsive design for mobile-first experience
* Touch-optimized interface elements
* Local data caching and synchronization
* Push notification support

## 4\. Design Preferences

### Interface

The interface should embody a clean, professional aesthetic with intuitive navigation optimized for mobile interactions. The design should prioritize clarity and ease of use while maintaining a trustworthy appearance suitable for financial management.

### Color Palette

* **Primary Color:** #2563EB (Professional Blue) - Main interface elements and CTAs
* **Secondary Color:** #64748B (Slate Gray) - Supporting text and secondary actions
* **Accent Color:** #10B981 (Success Green) - Payment confirmations and positive indicators
* **Alert Colors:** #EF4444 (Red) for overdue payments, #F59E0B (Amber) for warnings

### Typography

* **Headings:** Inter or Poppins - Modern, readable sans-serif for titles and headers
* **Body Text:** System fonts (San Francisco, Roboto) - Native system fonts for optimal mobile readability
* **Monospace:** JetBrains Mono - For financial figures and payment amounts

### Additional Design Considerations

* Touch-friendly button sizes (minimum 44px tap targets)
* High contrast ratios for accessibility
* Consistent spacing using 8px grid system
* Subtle shadows and rounded corners for modern appearance
* Loading states and micro-interactions for enhanced UX

## 6\. App Menu and Navigation Structure

### Primary Navigation (Bottom Tab Bar)

The app utilizes a bottom navigation bar optimized for thumb navigation with the following tabs:

* **Dashboard** (`/dashboard`) - Home icon - Main overview and quick stats
* **Tenants** (`/tenants`) - People icon - Tenant management and directory
* **Payments** (`/payments`) - Credit card icon - Payment tracking and collection
* **Reports** (`/reports`) - Chart icon - Financial analytics and exports
* **More** (`/settings`) - Menu icon - Settings and additional options

### Secondary Navigation

* **Header Navigation** \- Back arrows and contextual actions in top app bar
* **Floating Action Buttons** \- Quick access to primary actions \(Add Tenant\, Add Payment\)
* **Swipe Gestures** \- Left/right swipe on list items for quick actions
* **Pull-to-Refresh** \- Data refresh on main content screens

### Navigation Hierarchy

```
Dashboard (Root)
├── Tenants
│   ├── Tenant Details
│   ├── Add/Edit Tenant
│   └── Tenant Payment History
├── Payments
│   ├── Payment Details
│   ├── Add Payment
│   └── Edit Payment
├── Reports
│   └── Export Options
└── Settings
    ├── Profile Settings
    ├── Notification Preferences
    └── Data Management
```

### Deep Linking Support

Each screen supports deep linking for direct access and sharing:

* Tenant profiles: `/tenants/:tenantId`
* Payment records: `/payments/:paymentId`
* Reports with filters: `/reports?month=2024-01&tenant=123`

## 7\. User Flow

### Initial Setup Flow

1. **App Launch** \- User opens Renti PWA in mobile browser
2. **Authentication** \- User sees login screen\, either signs in or creates new account
3. **First-time Setup** \- After registration\, user is guided through app permissions \(notifications\, camera access\)
4. **Empty Dashboard** \- User lands on dashboard with empty state and onboarding tips
5. **Add First Tenant** \- Prominent call\-to\-action guides user to add their first tenant

### Primary User Journey - Rent Collection

1. **Dashboard Access** \- User opens app and sees dashboard with current month overview
2. **Payment Due Alert** \- User notices overdue payment notification banner
3. **Navigate to Payments** \- User taps bottom navigation "Payments" tab
4. **Filter Overdue** \- User filters payments by "Overdue" status to see pending collections
5. **Select Tenant** \- User taps on specific tenant with overdue payment
6. **Payment Details** \- User reviews payment details and tenant information
7. **Mark as Paid** \- User receives payment and marks it as paid\, selecting payment method
8. **Confirmation** \- System shows payment confirmation and updates dashboard totals
9. **Return to Dashboard** \- User navigates back to see updated statistics

### Tenant Management Flow

1. **Tenant Directory** \- User accesses tenant list from bottom navigation
2. **Search/Filter** \- User uses search to find specific tenant or browse all
3. **Tenant Selection** \- User taps on tenant card to view detailed profile
4. **Review Information** \- User checks contact details\, lease information\, and payment history
5. **Contact Tenant** \- User can directly call\, message\, or email tenant from profile
6. **Update Details** \- User edits tenant information if changes needed
7. **Payment History** \- User reviews tenant's complete payment history and trends

### Monthly Reporting Flow

1. **Reports Access** \- User navigates to Reports section from bottom navigation
2. **Date Selection** \- User selects month/quarter/year for report generation
3. **Data Review** \- User examines revenue charts\, payment distributions\, and trends
4. **Filter Analysis** \- User filters by specific tenants or payment methods for detailed insights
5. **Export Data** \- User exports report as PDF for records or tax purposes
6. **Share Results** \- User can share or print reports for accounting or management purposes

### Quick Payment Entry Flow

1. **Dashboard Quick Action** \- User taps floating "Add Payment" button from any screen
2. **Tenant Selection** \- User selects tenant from dropdown with type\-ahead search
3. **Payment Details** \- User enters amount\, date\, and payment method
4. **Receipt Upload** \- User optionally captures receipt photo using device camera
5. **Confirmation** \- User reviews details and saves payment
6. **Automatic Update** \- Dashboard and tenant records update automatically with new payment
7. **Notification** \- Optional: System sends payment confirmation to tenant

### Settings and Preferences Flow

1. **Settings Access** \- User taps profile avatar or "More" tab to access settings
2. **Profile Management** \- User updates personal information and app preferences
3. **Notification Setup** \- User configures payment reminders and alert preferences
4. **Data Management** \- User exports data\, sets up backups\, or imports existing tenant data
5. **Support Access** \- User accesses help documentation or contact support if needed

### Offline Usage Flow

1. **Offline Detection** \- App detects lost internet connection and shows offline indicator
2. **Local Data Access** \- User can still view cached tenant and payment data
3. **Offline Entry** \- User can add payments and tenant updates\, stored locally
4. **Sync Queue** \- App queues changes for synchronization when connection restored
5. **Auto Sync** \- When online\, app automatically syncs all pending changes to server
6. **Conflict Resolution** \- If conflicts occur\, app presents options to user for resolution
